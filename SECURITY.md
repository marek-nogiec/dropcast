# Security policy

## Supported versions

The latest release and the current `main` branch receive security fixes. Older
releases are not supported.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Once this repository
is public, use **Security → Report a vulnerability** on GitHub to send a private
report. Include affected versions, reproduction steps, impact, and any proposed
mitigation.

While private vulnerability reporting is unavailable before publication,
contact [@marek-nogiec](https://github.com/marek-nogiec) without sensitive
details and request a private communication channel.

Please allow reasonable time to investigate and prepare a fix before public
disclosure.
