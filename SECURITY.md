# Security policy

## Supported versions

The latest release and the current `main` branch receive security fixes. Older
releases are not supported.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Use
**Security → Report a vulnerability** on GitHub to send a private report.
Include affected versions, reproduction steps, impact, and any proposed
mitigation.

Please allow reasonable time to investigate and prepare a fix before public
disclosure.
